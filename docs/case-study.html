Case StudiesÔÉÅ

This section provides two case studies demonstrating how to use STRmie-HD depending on the sequencing design:

  1. Paired-End (PE) reads ‚Äì combine R1/R2 into single continuous reads using an external merging tool, then run STRmie-HD.

  2. Single-End (SE) reads ‚Äì Unpaired reads (Single-End), such as Illumina or long reads from Nanopore/PacBio. Run STRmie-HD directly on gzipped FASTQ/FASTA files.

Both flows converge to the same STRmie-HD pipeline and produce the same interactive HTML report for visual inspection.

Case Study 1 ‚Äì Paired-End Reads (PE) with PEARÔÉÅ

When working with paired-end data, we recommend merging reads upstream with PEAR to obtain single, high-confidence sequence spanning the CAG repeat region.

Generic PEAR command

pear -f forward_R1.fastq.gz -r reverse_R2.fastq.gz -v 10 -o output_prefix

Parameters

  • -f forward_R1.fastq.gz ‚Üí path to the forward read (R1).

  • -r reverse_R2.fastq.gz ‚Üí path to the reverse read (R2).

  • -v MIN_OVERLAP ‚Üí minimum overlap length (in bp) required to confidently merge a pair (default: 10).

  • -o output_prefix ‚Üí output filename prefix; PEAR will produce files such as output_prefix.assembled.fastq (merged reads).

Alternative merging tool: FLASH (Fast Length Adjustment of Short reads) can also be used for merging paired-end reads before running STRmie-HD.

After merging, use the merged reads as input for STRmie‚ÄëHD (see Running the Complete Pipeline below).

Case Study 2 ‚Äì Single-End Reads (SE)ÔÉÅ

For single-end datasets, no merging is required. STRmie‚ÄëHD can ingest fastq.gz (preferred, includes quality) or fasta.gz files format directly.

Running the Complete Pipeline (applies to both PE-merged and SE)ÔÉÅ

Launch STRmie‚ÄëHD with default parameters on your input directory (either the PE‚Äëmerged .assembled.fastq outputs or the original SE files):

strmie --mode Complete_Pipeline \
       -f /path/to/input_dir \
       -o /path/to/output_dir \
       [other options]

required parameters

  • -f /path/to/input_dir/ ‚Üí directory containing input reads (e.g., merged *.assembled.fastq from PEAR, or SE .fastq.gz / .fasta.gz).

  • -o /path/to/output_dir/ ‚Üí output directory for all STRmie‚ÄëHD outputs.
